# بوت البحث عن الصفقات — دليل الاستخدام

بوت يقرأ قائمة المتابعة (watchlist) المُصدّرة من TradingView، ويبحث تلقائياً في
الأسهم الأمريكية والعملات الرقمية، ويحلّلها باستراتيجية فنية متعددة العوامل،
ثم يعطيك إشارات دخول/خروج مرتّبة مع الأسباب ووقف الخسارة والهدف.

> ⚠️ **تنبيه:** هذه أداة تحليل وتعليم فقط، وليست نصيحة مالية. البوت **لا ينفّذ أي صفقة**
> ولا يتصل بأي حساب وساطة أو حساب TradingView. قرار التداول ومسؤوليته عليك وحدك.
> التداول ينطوي على مخاطر حقيقية لخسارة رأس المال.

---

## لماذا لا يتصل مباشرة بحساب TradingView؟

TradingView لا توفّر واجهة برمجية رسمية (API) تتيح قراءة حسابك أو قائمتك تلقائياً.
الحل العملي الذي يستخدمه هذا البوت: **تصدير قائمتك** من TradingView كملف نصّي ثم
قراءته. البوت يفهم تنسيق `EXCHANGE:SYMBOL` مباشرةً (نفس ملفك الحالي).

لتحديث القائمة لاحقاً: في TradingView افتح الـ watchlist ← القائمة (⋮) ← Export،
واستبدل ملف `watchlist.txt`.

## التثبيت (مرة واحدة)

تحتاج Python 3.9 أو أحدث، ثم:

```bash
pip install -r requirements.txt
```

## التشغيل

```bash
python3 trading_bot.py --watchlist watchlist.txt
```

سيفحص البوت كل الرموز ويطبع أفضل الفرص، ويحفظ تقريراً كاملاً باسم
`signals_YYYYMMDD_HHMM.csv` يمكن فتحه في Excel.

### خيارات مفيدة

| الخيار | الوصف | الافتراضي |
|---|---|---|
| `--assets all\|crypto\|stocks` | فحص الكل أو الكريبتو فقط أو الأسهم فقط | `all` |
| `--timeframe 1d\|4h\|1h` | الإطار الزمني للتحليل | `1d` |
| `--top N` | كم فرصة تُعرض في الملخص | `25` |
| `--min-score N` | أقل درجة لاعتبارها إشارة | `20` |
| `--output-dir PATH` | مجلد حفظ التقرير | المجلد الحالي |

أمثلة:

```bash
# الكريبتو فقط على إطار 4 ساعات
python3 trading_bot.py --watchlist watchlist.txt --assets crypto --timeframe 4h

# أقوى 10 فرص فقط بدرجة 30 فأعلى
python3 trading_bot.py --watchlist watchlist.txt --top 10 --min-score 30
```

## كيف يفكّر البوت؟ (الاستراتيجية)

لكل رمز يحسب مجموعة مؤشرات ويجمعها في **درجة** من ‎-100‎ إلى ‎+100‎:

- **الاتجاه:** موقع السعر من EMA50 و EMA200، وتقاطع EMA20/EMA50.
- **الزخم:** اتجاه هيستوجرام MACD (إيجابي/سلبي ومتصاعد/متراجع).
- **القوة النسبية RSI:** صحي (50–65) إيجابي، تشبّع شرائي (>75) تحذيري، تشبّع بيعي (<30) ارتداد محتمل.
- **الحجم:** ارتفاع الحجم فوق متوسطه مع اتجاه السعر.
- **الاختراق:** قرب قمة / قاع آخر 20 شمعة (مكافأة مخفّضة لتفادي مطاردة القمة).
- **جودة الدخول (جديد):** يقيس بُعد السعر عن EMA20 بوحدات ATR — يكافئ الدخول قرب
  المتوسط (ارتداد صحي) ويعاقب السعر الممتد بعيداً (مطاردة الحركة = دخول رديء).
- **الدايفرجنس:** انحراف إيجابي/سلبي بين السعر و RSI(21) أو MACD (مقارنة آخر قاعين/قمتين).

ثم يصنّف الإشارة: **شراء قوي / شراء / حيادي / بيع / بيع قوي**.

### وقف الخسارة الهيكلي (جديد)
بدل وقف ثابت على التذبذب، يضع البوت الوقف **تحت آخر قاع حقيقي** (swing low) ضمن آخر
40 شمعة + هامش ‎0.5×ATR‎ (للشراء؛ وفوق آخر قمة للبيع). مع ضوابط: حدّ أقصى للمخاطرة
‎3.5×ATR‎ (يمنع خسارة ضخمة) وحدّ أدنى ‎1.0×ATR‎ (يمنع الضرب بالضجيج). ثم يُعاد حساب
الأهداف والعائد/المخاطرة على الوقف الجديد. الهدف: تقليل ضرب الوقف بالحركات العشوائية.

### طريقة الأهداف
- `--targets fib` (الافتراضي): **امتدادات فيبوناتشي** بنسب 0.618 و1.0 و1.618
  محسوبة من نطاق آخر حركة سعرية (swing) خلال آخر 50 شمعة.
- `--targets atr`: أهداف بمضاعفات ATR عند 1R / 2R / 3R.

إن كان نطاق الـ swing ضيقاً جداً، يتحوّل البوت تلقائياً لطريقة ATR لتفادي أهداف غير منطقية.

### فلتر اتجاه السوق العام 🌐
يحدّد البوت اتجاه السوق مرة واحدة لكل تشغيل: **BTC** للكريبتو و**S&P 500 (SPY)** للأسهم
(إطار يومي، مقارنة بـ EMA50/EMA200). يظهر اتجاه السوق في كل بطاقة، ويُعتبر جزءاً من
شروط «الصفقة عالية القناعة» (لا قناعة عالية إن كانت الصفقة معاكسة للسوق).

- **الوضع الافتراضي (مرن):** يُعرض اتجاه السوق فقط، وكل الإشارات تُرسل — لتتمكّن من المقارنة.
- **الوضع الصارم:** أضف `--market-filter` لاستبعاد كل صفقة معاكسة للسوق:
```bash
python3 trading_bot.py --watchlist watchlist.txt --market-filter
```

### المتوسط السنوي (365 يوم) 📅
يُحسب من بيانات يومية (365 شمعة للكريبتو، 252 للأسهم ≈ سنة تداول). يظهر في كل بطاقة
سطر: `📅 متوسط 365 يوم: فوقه ✅` أو `تحته ⚠️` كعامل قوة طويل المدى.

وهناك **تنبيه مستقل متعدد الفريمات** عند اختراق السعر للمتوسط السنوي:
```bash
python3 trading_bot.py --mode yearly --watchlist watchlist.txt
```
المتوسط قيمة واحدة (من بيانات يومية)، لكن البوت يفحص لحظة عبوره على **اليومي + 4 ساعات + ساعة**
(للكريبتو؛ الأسهم يومي فقط) ويوضّح في التنبيه على أي فريم حدث الاختراق ووضع كل فريم:
```
🔭 اخترق على الفريمات: 1h
📊 الوضع الحالي: 1d تحت ⚠️ | 4h تحت ⚠️ | 1h فوق ✅
```
الساعة تكشف العبور أبكر، واليومي يؤكّده. يمنع التكرار لكل فريم على حدة عبر `yearly_cross.json`.

> لالتقاط عبور 1h/4h داخل اليوم، شغّل وضع `yearly` بتكرار أعلى (مثلاً كل ساعة). التشغيل
> اليومي الواحد يكفي لعبور الفريم اليومي فقط.

### وسم «الصفقة عالية القناعة ⭐»
تظهر الصفقة بوسم **⭐⭐ عالية القناعة ⭐⭐** عند اجتماع ثلاثة شروط معاً:
درجة قوية (≥ 40)، وتوافق على فريمين أو أكثر، ووجود دايفرجنس إيجابي **أو** زخم MACD صاعد.
هذه الصفقات تُرسل أولاً وتُرتّب في المقدمة. باقي الصفقات تبقى إشارات عادية للمراجعة.

### تصفية الدايفرجنس (اختياري)
لإرسال صفقات الشراء التي بها **دايفرجنس إيجابي مؤكّد فقط**:
```bash
python3 trading_bot.py --watchlist watchlist.txt --require-divergence
```

> يمكنك تعديل أوزان العوامل والعتبات بسهولة داخل دالة `analyze()` في `trading_bot.py`.

## إرسال الإشارات إلى تيليجرام (اختياري)

يستطيع البوت إرسال أفضل الفرص إلى محادثتك في تيليجرام بعد كل فحص.

**الخطوة 1 — أنشئ بوتاً واحصل على التوكن:**
في تيليجرام ابحث عن `@BotFather` ← أرسل `/newbot` ← اختر اسماً ← ستحصل على
**توكن** بهذا الشكل: `123456789:ABCdef...`.

**الخطوة 2 — احصل على معرّف المحادثة (chat id):**
أرسل أي رسالة لبوتك الجديد، ثم افتح في المتصفح (ضع توكنك مكان `<TOKEN>`):
```
https://api.telegram.org/bot<TOKEN>/getUpdates
```
ابحث عن `"chat":{"id":XXXXXXXX` — هذا الرقم هو معرّف محادثتك.

**الخطوة 3 — شغّل البوت مع تيليجرام:**
```bash
python3 trading_bot.py --watchlist watchlist.txt \
  --telegram-token 123456789:ABCdef... \
  --telegram-chat-id 12345678
```

أو بدلاً من كتابتهما في كل مرة، ضعهما كمتغيّرات بيئة:
```bash
export TELEGRAM_TOKEN=123456789:ABCdef...
export TELEGRAM_CHAT_ID=12345678
python3 trading_bot.py --watchlist watchlist.txt
```

## متابعة الصفقات المفتوحة (الأهداف ووقف الخسارة)

عند إرسال إشارة، يحفظها البوت في `open_trades.json`. ثم وضع **المتابعة** يفحص
السعر اللحظي ويرسل تحديثاً عند تحقق كل هدف (الأول/الثاني/الثالث) أو ضرب وقف الخسارة.

تشغيل المتابعة يدوياً:
```bash
python3 trading_bot.py --mode monitor
```

لتشغيلها تلقائياً كل 30 دقيقة (cron) — بعد وضع التوكن في `run_monitor.sh` و`chmod +x`:
```bash
*/30 * * * * /مسار/المجلد/run_monitor.sh
```

> الفرق: `--mode scan` يبحث عن صفقات جديدة، و`--mode monitor` يتابع المفتوحة منها.

## الاختبار التاريخي (Backtest) 🔬

قبل الوثوق بأي تعديل، اختبره على الماضي. يعيد البوت تشغيل قائمتك شمعةً بشمعة
(بدون نظر للمستقبل)، يفتح صفقات افتراضية بنفس منطق الإشارات، ويقيس النتيجة
بوحدات المخاطرة (R). يقارن سيناريوهين لنفس الدخولات:

- **بدون إدارة:** مسك الصفقة حتى الهدف الأخير أو الوقف.
- **مع إدارة:** جني 50% عند الهدف الأول + نقل الوقف لنقطة الدخول (breakeven).

```bash
# اختبار الكريبتو على آخر سنة
python3 trading_bot.py --mode backtest --watchlist watchlist.txt --assets crypto --bt-bars 365

# تشديد الجودة (درجة أعلى) + فلتر السوق
python3 trading_bot.py --mode backtest --watchlist watchlist.txt --min-score 51
```

يطبع التقرير: **عدد الصفقات، نسبة الربح، التوقّع (متوسط R — الأهم)، معامل الربح،
أقصى تراجع، أطول سلسلة خسائر، ونسبة ضرب الوقف**، ويحفظ كل الصفقات في ملف
`backtest_*.csv`.

> 💡 المقياس الفاصل هو **التوقّع (متوسط R)** لا نسبة الربح. بعائد/مخاطرة جيد قد يكون
> النظام رابحاً حتى لو ضُرب الوقف في أغلب الصفقات. نتائج تاريخية لا تضمن المستقبل.

## ملاحظات على قائمتك

- الأسهم الأمريكية (NASDAQ/NYSE/AMEX) والعملات (أزواج USDT) مدعومة بالكامل.
- الرموز الكلية (BTC.D, DXY, SPX, VIX, الذهب) تُستخدم كسياق عام ولا تُولَّد لها إشارات.
- الأسهم الإماراتية (ADX/DFM) لا تتوفّر لها بيانات مجانية موثوقة، لذا تُتجاهَل تلقائياً.
