# تشغيل البوت في السحابة مجاناً (GitHub Actions)

الهدف: يعمل البوت تلقائياً مرتين يومياً على خوادم GitHub المجانية ويرسل الإشارات
إلى تيليجرام على جوالك — **دون الحاجة لتشغيل الماك أو الجوال**.

> 🔒 مهم للأمان: لا نضع التوكن داخل أي ملف. نضعه في "Secrets" المشفّرة، ونجعل
> المستودع **خاصاً (Private)**. لا ترفع ملف `run_daily.sh` (يحتوي التوكن).

---

## ما الذي سترفعه؟
أربعة ملفات فقط:
- `trading_bot.py`
- `watchlist.txt`
- `requirements.txt`
- `scan.yml`  ← يوضع في مسار خاص: `.github/workflows/scan.yml`

---

## الخطوات

### 1) أنشئ حساب GitHub
افتح <https://github.com> ← Sign up (مجاني).

### 2) أنشئ مستودعاً جديداً (Private)
اضغط **+** أعلى اليمين ← **New repository** ← اسم مثل `trade-bot` ← اختر
**Private** ← **Create repository**.

### 3) ارفع الملفات الثلاثة الأساسية
في صفحة المستودع: **Add file ← Upload files** ← اسحب
`trading_bot.py` و `watchlist.txt` و `requirements.txt` ← **Commit changes**.

### 4) أضف ملف الجدولة في مساره الصحيح
اضغط **Add file ← Create new file**، واكتب في خانة الاسم بالضبط:
```
.github/workflows/scan.yml
```
(كتابة `/` تنشئ المجلدات تلقائياً). ثم الصق محتوى ملف `scan.yml` كاملاً ←
**Commit changes**.

### 5) أضف التوكن والمعرّف كـ Secrets
في المستودع: **Settings ← Secrets and variables ← Actions ← New repository secret**.
أضف سرّين:

| الاسم (Name) | القيمة (Value) |
|---|---|
| `TELEGRAM_TOKEN` | توكن بوتك (الجديد بعد التجديد) |
| `TELEGRAM_CHAT_ID` | `1221410554` |

### 6) جرّبه يدوياً الآن
تبويب **Actions** ← اختر **trade-scan** ← زر **Run workflow** ← **Run workflow**.
بعد دقيقتين ستصلك رسالة على تيليجرام. ✅

بعدها سيعمل تلقائياً كل يوم:
- **08:00 ص (الإمارات)** فحص كريبتو، إطار 4 ساعات.
- **04:00 م (الإمارات)** فحص أسهم، إطار يومي.

---

## متابعة الصفقات في السحابة
ملف `scan.yml` يتضمّن أيضاً **متابعة تلقائية كل ساعتين**: يفحص الصفقات المفتوحة
ويرسل تحديثاً عند تحقق هدف أو ضرب وقف خسارة. لكي تبقى حالة الصفقات بين التشغيلات،
يحفظ الـ workflow ملف `open_trades.json` داخل المستودع تلقائياً (لذلك يحتاج صلاحية
`contents: write` الموجودة في الملف — لا تحذفها).

> للمتابعة الأكثر تكراراً (مثلاً كل 30 دقيقة): اجعل المستودع **عاماً (Public)** للحصول
> على دقائق تشغيل غير محدودة (الأسرار تبقى مخفية حتى في المستودع العام)، ثم غيّر
> سطر `'0 */2 * * *'` إلى `'*/30 * * * *'`.

---

## ملاحظات
- المواعيد في ملف `scan.yml` بتوقيت UTC (الإمارات = UTC+4). لتغييرها عدّل سطور `cron`.
- قد تتأخر مواعيد GitHub بضع دقائق في أوقات الذروة — أمر طبيعي.
- الباقة المجانية تكفي بكثير لتشغيلين يومياً.
- لتحديث قائمتك لاحقاً: استبدل `watchlist.txt` في المستودع (Upload files فوق القديم).
- إن أردت إضافة/تعديل المواعيد أو الأصول، عدّل `scan.yml`.

> ⚠️ تذكير: أداة تحليل تعليمية فقط — ليست نصيحة مالية. البوت لا ينفّذ صفقات.
